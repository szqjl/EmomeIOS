
2. **查看 Actions**：
   - 点击仓库顶部的 "Actions" 标签页
   - 查看 "iOS Build" 和 "Flutter Test" 工作流状态
   - 首次推送应该会自动触发构建

3. **等待构建完成**：

构建完成后：

1. 进入 Actions 页面
2. 选择已完成的构建任务
3. 滚动到页面底部
4. 在 "Artifacts" 部分下载 `ios-build-artifacts`

### 3. 手动触发构建（可选）

如果需要手动触发构建：

1. 进入 GitHub 仓库
2. 点击 "Actions" 标签页
3. 选择 "iOS Build" 工作流
4. 点击 "Run workflow" 按钮
5. 选择分支并运行

## 🐛 故障排除

### GitHub Actions 未触发

**问题**：推送代码后 Actions 未自动运行

**解决方案**：
1. 检查 `.github/workflows/` 目录是否存在
2. 检查工作流文件语法是否正确
3. 在 Actions 页面手动触发一次

### 构建失败

**问题**：构建任务失败

**解决方案**：
1. 查看构建日志，找到错误信息
2. 检查 Flutter 版本是否匹配
3. 检查依赖是否正确
4. 查看 GitHub Actions 使用指南

### 权限问题

**问题**：无法推送到 GitHub

**解决方案**：
- 使用 HTTPS 方式（已配置）
- 或配置 SSH 密钥：
  ```bash
  ssh-keygen -t ed25519 -C "your_email@example.com"
  # 将公钥添加到 GitHub Settings > SSH and GPG keys
  ```
